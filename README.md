# 豆瓣电影 Top250 大数据统计分析

## 项目简介
独立搭建 Hadoop 完全分布式集群与 HBase 数据库环境，完成从数据上传、分布式存储、统计计算到结果落地的完整数据处理链路。本项目为《大数据分布式系统框架》结课实验。

## 技术栈
- **分布式存储**：Hadoop HDFS 3.1.3
- **分布式数据库**：HBase 1.2.6
- **计算框架**：MapReduce（Java API）
- **编程语言**：Java
- **环境**：Linux (Ubuntu 18.04)

## 项目流程
1. **环境搭建**：独立安装并配置 Hadoop 3.1.3 与 HBase 1.2.6，解决组件间依赖冲突。
2. **数据上传**：将豆瓣电影 Top250 数据集通过 Shell 命令上传至 HDFS。
3. **数据入库**：编写 Java 程序，通过 HBase API 将 HDFS 中的数据导入原始数据表 `douban_movie_raw`。
4. **统计计算**：编写 Java 程序 `YearCountSimple`，使用 Scan 扫描全表并结合 HashMap 按上映年份统计电影数量。
5. **结果存储**：将统计结果写入 HBase 结果表 `movie_analysis_result`。

## 难点与解决
- **Guava 版本冲突**：Hadoop 与 HBase 的 Guava 版本不兼容，导致统计程序 12 分钟无响应。尝试替换 jar 包、调整 classpath 均未解决，最终通过升级 HBase 版本并清理 ZooKeeper 残留元数据彻底修复。
- **权限问题**：HBase 目录所有者权限配置错误导致启动失败，通过分析日志定位根因并修改目录权限解决。
- **环境可复现性**：反复遇到同类配置问题后，养成每次修改后验证的习惯，确保环境稳定可用。

## 项目收获
- 掌握了 Hadoop 与 HBase 的协同工作流程及 Java API 的使用。
- 深刻理解大数据组件间版本兼容的重要性，学会通过日志定位“不报错的软故障”。
- 积累了独立搭建和调试分布式环境的工程实践经验。
